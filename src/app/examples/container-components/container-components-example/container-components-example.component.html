Container komponenty

/app/examples/container-components

Máme komponent ktorého state je reprezentovaný ako interface Person. Vo výsledku je to objekt ktorý má hodnoty name, surname a age.

Tento komponet má inštanciu state uloženú v premennej state. Tento state je renderovaný do textovej podoby vo svetlom vnútre komponentu cez interpolation binding.

Pomocou tlačidiel môžeme meniť state komponentu a sledovať, či sa vnútorná zmena state prejaví aj v zobrazených údajoch.
Tlačidlá pre oneskorenú zmenu majú v sebe obyčajný setTimeout s 500ms, ale vo svojej podstate je to to isté, ako keby prišla nová hodnota stavu cez nejaký async call.

Tento prvý komponent má nastavenú Change Detection Strategy na Default hodnotu...

Všetko funguje ako má, aj oneskorená zmena stavu sa prejaví.

Teraz sa ale pozrime, ako sa v takomto istom scenári zachová komponent s Change Detection Strategy nastavenou na On Push...
